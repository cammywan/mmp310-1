JavaScript datatypes

MMP 310 > Week 3

Last week we introduced the concept of a variable, an object that stores a piece of information. Today we will look deeper at data types and how they are used in JavaScript.

Variables can be different data types. The basic types in JavaScript are:

  • number
  • string
  • boolean
  • array

Later in the semester we will talk about objects and functions.

Numbers

var x = 1;
console.log(x);
x++;
console.log(x);
typeof x;

In JavaScript does not differentiate between integers and floating point numbers, such as 1 and 1.5.

JavaScript can also convert strings that have only numeric characters into numbers for us, which is a nice thing for new programmers, but can get a little hairy.

NaN means "Not a Number". You will get that if you try to evaluate a string that cannot be parsed to a number. You can test for NaN using the isNaN() function.

 isNaN(NaN);

Strings

Strings in JavaScript are sequences of characters, using Unicode they are represented by a 16-bit number. Characters are written as strings with the length of one. Strings provide length, a property that returns the length of the string.

"Hello World".length

Strings also have some built in methods like charAt() and replace() which help us do evaluation and manipulation.

"Hello World".charAt(0);
"Hello World".replace("World", "Jerry");

Booleans

Booleans default to the values true and false. In JavaScript many values will become false, such as 0, "" (empty string), NaN, null, and undefined. Everything else defaults to true, such as "Hello World", 1, 99, or -99. Try evaluating some booleans in the console with this syntax:

Boolean("");
Boolean(null);
Boolean(4);

We'll use booleans when we start working on logic.

Arrays

An array is an object that contains a list of variables, such as this:

var imgs = ["cat.jpg", "dog.jpg", "bird.jpg"];

In JavaScript, the items in an array can have any datatype, like a number, string, boolean, or even another arrary. In other languages, the data in an array usually are all of the same type. The elements of an array are accessed by their index number, starting with zero.

imgs[0] will return "cat.jpg". Like strings, arrays have a length property. Try creating an array, reading from the list, and returning the length:

myArray = ["cat", "dog", "alligator"];
myArray.length;
myArray[0];
myArray[2];

Adding elements to an array

imgs = ["cat", "dog", "bird"];
imgs[3] = "alligator";
imgs.push("lion");
imgs.pop();

Null and undefined

JavaScript uses null and undefined as distinct values. null indicates a purposely unassigned value. undefined indicates a variable that has been instantiated but not assigned with a value.

console.log(y);;
var x;
console.log(x);

Testing type

When using an array, we can assign multiple types of information in one array, so it can be useful to test the type of the array before using the data.

var a = ["hello", "world", 0, 1, 2, 3];
console.log(typeof a[0]);
console.log(typeof a[4]);
console.log(typeof a[a.length - 1]);

Array methods

Arrays have some useful methods for manipulation.

var a = ["h", "e", "l", "l", "o"];
var b = a.join("");
console.log(typeof b);
var c = b.split("");
console.log(typeof c);
Array.isArray(c);
